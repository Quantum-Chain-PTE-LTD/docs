This guide walks you through a complete integration flow: authenticate, create organizational resources, and submit your first transaction on Quantum Chain.

## Prerequisites

- A running Custody API instance — see [Development](/development) for local setup
- An API key pair — contact the platform team or use the admin CLI

<Note>
All examples use `http://localhost:8080` as the base URL. Replace with your deployment URL in production.
</Note>

## Step 1: Authenticate

Every request requires a Bearer token in the format `key_id:secret`.

<CodeGroup>
```bash cURL
export CUSTODY_API_KEY="your_key_id:your_secret"
export BASE_URL="http://localhost:8080/v1"

# Test credentials
curl -s "$BASE_URL/healthz" | jq .
```

```python Python SDK
from quantumchain_custody import Client

client = Client(
    api_key="your_key_id:your_secret",
    base_url="http://localhost:8080"
)
health = client.health.check()
print(health)
```
</CodeGroup>

```json
{
  "status": "healthy"
}
```

## Step 2: Create a tenant

A tenant is an isolated organizational unit. All vaults, wallets, and transactions belong to a tenant.

<CodeGroup>
```bash cURL
curl -s -X POST "$BASE_URL/tenants" \
  -H "Authorization: Bearer $CUSTODY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "My Organization"}' | jq .
```

```python Python SDK
tenant = client.tenants.create(name="My Organization")
print(f"Tenant ID: {tenant.id}")
```
</CodeGroup>

```json
{
  "id": "tnt_abc123",
  "name": "My Organization",
  "status": "active",
  "created_at": "2026-03-19T10:00:00Z"
}
```

Save your tenant ID:

```bash
export TENANT_ID="tnt_abc123"
```

## Step 3: Create a vault account

A vault account groups related wallets together.

```bash
curl -s -X POST "$BASE_URL/vault/accounts" \
  -H "Authorization: Bearer $CUSTODY_API_KEY" \
  -H "Content-Type: application/json" \
  -d "{\"name\": \"Treasury\", \"tenant_id\": \"$TENANT_ID\"}" | jq .
```

```json
{
  "id": "va_def456",
  "name": "Treasury",
  "tenant_id": "tnt_abc123",
  "created_at": "2026-03-19T10:01:00Z"
}
```

## Step 4: Register a wallet

Register an external Quantum Chain wallet address. The custody API does not generate keys — you provide the Dilithium3 public key and address from your external signer.

```bash
curl -s -X POST "$BASE_URL/vault/accounts/va_def456/wallets" \
  -H "Authorization: Bearer $CUSTODY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "address": "0x1234...abcd",
    "public_key": "<base64-encoded-dilithium3-public-key>",
    "name": "Hot Wallet"
  }' | jq .
```

```json
{
  "id": "wlt_ghi789",
  "address": "0x1234...abcd",
  "vault_account_id": "va_def456",
  "name": "Hot Wallet",
  "created_at": "2026-03-19T10:02:00Z"
}
```

## Step 5: Create a transaction

Submit a transfer. The transaction enters `SIGNING_REQUESTED` status and waits for your external signer.

<CodeGroup>
```bash cURL
curl -s -X POST "$BASE_URL/transactions" \
  -H "Authorization: Bearer $CUSTODY_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: first-transfer-001" \
  -d '{
    "asset_id": "QC_NATIVE",
    "source": {
      "type": "VAULT_ACCOUNT",
      "id": "va_def456"
    },
    "destination": {
      "type": "ONE_TIME_ADDRESS",
      "one_time_address": {
        "address": "0x5678...efgh"
      }
    },
    "amount": "1.5",
    "note": "First transfer"
  }' | jq .
```

```python Python SDK
tx = client.transactions.create(
    source_wallet_id="wlt_ghi789",
    destination_addr="0x5678...efgh",
    asset_id="QC_NATIVE",
    amount="1.5",
    note="First transfer",
    idempotency_key="first-transfer-001"
)
print(f"Transaction ID: {tx.id}, Status: {tx.status}")
```
</CodeGroup>

```json
{
  "id": "tx_jkl012",
  "status": "SIGNING_REQUESTED",
  "asset_id": "QC_NATIVE",
  "amount": "1.5",
  "created_at": "2026-03-19T10:03:00Z"
}
```

## Step 6: Sign the transaction

Retrieve the signing payload and submit the Dilithium3 signature from your external signer.

<CodeGroup>
```bash cURL
# Get the unsigned digest
curl -s "$BASE_URL/transactions/tx_jkl012/signing_payload" \
  -H "Authorization: Bearer $CUSTODY_API_KEY" | jq .

# Sign the hash in your HSM/key vault, then submit:
curl -s -X POST "$BASE_URL/transactions/tx_jkl012/signature" \
  -H "Authorization: Bearer $CUSTODY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "signature": "<base64-encoded-dilithium3-signature>",
    "signer_public_key": "<base64-encoded-public-key>"
  }' | jq .
```

```python Python SDK
# Get the signing payload
payload = client.transactions.get_signing_payload("tx_jkl012")
print(f"Signing hash: {payload.signing_hash}")

# Sign in your HSM/key vault, then submit:
# signature_hex, public_key_hex = your_signer(payload.signing_hash)
result = client.transactions.submit_signature(
    "tx_jkl012",
    signature=signature_hex,
    public_key=public_key_hex
)
print(f"Status: {result.status}")
```
</CodeGroup>

The transaction moves through `SIGNED` → `BROADCASTING` → `CONFIRMING` → `COMPLETED`.

## Step 7: Verify completion

Poll the transaction or set up a [webhook](/concepts/webhooks) to receive status updates.

<CodeGroup>
```bash cURL
curl -s "$BASE_URL/transactions/tx_jkl012" \
  -H "Authorization: Bearer $CUSTODY_API_KEY" | jq .status
```

```python Python SDK
tx = client.transactions.get("tx_jkl012")
print(f"Status: {tx.status}")
```
</CodeGroup>

```json
"COMPLETED"
```
